# Quick Workflows

This page gives fast, practical usage patterns.

## Workflow 1: Daily planning in under 2 minutes

1. Open calendar.
2. Click empty slots to create your top 3 events.
3. Drag to rebalance your day.
4. Use category naming for quick color grouping.

See: [Event Management](../events/manage.md), [Category Coloring](../events/categories.md)

## Workflow 2: Mixed local and remote setup

1. Use Full Note or Daily Note for editable personal planning.
2. Add remote ICS/CalDAV/Google calendars for external schedules.
3. Keep one local calendar as your default write target.

See: [Calendar Types](../calendars/local.md), [ICS](../calendars/ics.md), [CalDAV](../calendars/caldav.md), [Google](../calendars/gcal.md)

## Workflow 3: Weekly operating workspace

1. Create a workspace focused on your current project calendars.
2. Save a preferred view and date behavior.
3. Switch workspaces as context changes.

See: [Workspaces](../views/workspaces.md)

## Workflow 4: Task scheduling with backlog

1. Open tasks backlog (if configured).
2. Drag task to a date/time slot.
3. Adjust duration using drag/resize.

See: [Tasks Plugin Integration](../calendars/tasks-plugin-integration.md)

## Workflow 5: Timezone-safe planning

1. Set display timezone intentionally.
2. Verify cross-calendar events around DST transitions.
3. Keep source calendars authoritative for source timezone.

See: [Timezone Support](../events/timezones.md)
